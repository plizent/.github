# Governance

This document describes how Plizent is governed and how organization-wide decisions are made.

It defines the roles and responsibilities of contributors, maintainers, and organization administrators, and explains the processes used to guide the project's development and community.

All members of the community are expected to follow our [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

---

## Mission & Principles

Plizent is committed to building reliable, open-source software through open collaboration and responsible engineering.

Our governance is guided by the following principles:

- Make decisions openly whenever possible.
- Encourage respectful and constructive collaboration.
- Prioritize correctness, maintainability, and long-term sustainability.
- Keep organization-wide policies consistent across repositories.
- Welcome contributions from everyone, regardless of experience.

---

## Governance Scope

This document defines the default governance model for Plizent repositories.

It applies to:

- Organization-wide policies and standards.
- Community health documentation.
- Technical governance and decision-making.
- Maintainer roles and responsibilities.
- Repository management and administration.

This document does **not** define:

- Project roadmaps.
- Feature prioritization.
- Repository-specific architecture or implementation details.
- Deployment, operations, or release procedures.

Repositories may provide additional governance or project-specific documentation. When available, follow the repository's documentation for repository-specific processes.

---

## Roles & Responsibilities

### Contributors

Contributors help improve Plizent by:

- Reporting bugs and security issues.
- Suggesting new features and improvements.
- Improving documentation.
- Submitting pull requests.
- Participating in discussions and code reviews.

### Maintainers

Maintainers are responsible for:

- Reviewing and merging contributions.
- Maintaining code quality and documentation.
- Enforcing community standards and project policies.
- Managing releases and repository maintenance.
- Helping contributors and guiding technical discussions.

### Organization Administrators

Organization administrators are responsible for:

- Managing organization settings and repository access.
- Maintaining organization-wide policies and security settings.
- Managing teams, permissions, and repository ownership.
- Responding to organization-level security or administrative matters.

---

## Decision-Making Process

Plizent values open, transparent, and collaborative decision-making.

Whenever possible, technical discussions and decisions take place through GitHub Issues, Discussions, and Pull Requests so they are visible to the community.

Maintainers work to build consensus by considering technical quality, long-term maintainability, project goals, and community feedback.

When consensus cannot be reached, the maintainers make the final decision in the best interest of the project.

Security-related decisions may follow a private process until the issue has been resolved, as described in [SECURITY.md](SECURITY.md).

---

## Proposal Process

We welcome ideas for improving Plizent.

Most changes can be proposed by opening an issue, starting a discussion, or submitting a pull request.

For significant organization-wide changes, we recommend the following process:

1. Open an issue or discussion to explain the proposal.
2. Gather feedback from the community and maintainers.
3. Submit a pull request with the proposed changes.
4. Address review feedback and update the proposal as needed.
5. Merge the changes after the required approvals have been received.

Following this process helps encourage open discussion and informed decision-making before changes are adopted.

---

## Repository Governance

This document provides the default governance model for Plizent repositories.

Individual repositories may define additional policies, workflows, coding standards, or project-specific governance to meet their own requirements.

When repository-specific governance documentation is available, follow that documentation for repository-specific processes while continuing to follow the organization-wide policies defined in this repository.

---

## Changes to This Governance

Governance evolves as Plizent and its community grow.

Changes to this document should be proposed through a pull request that clearly explains:

- The purpose of the proposed change.
- The reason the change is needed.
- Any expected impact on contributors, maintainers, or repositories.

Significant governance changes should allow time for community discussion and maintainer review before they are adopted.
